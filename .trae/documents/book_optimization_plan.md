# 书籍创建流程与界面优化计划

## 1. 项目现状分析

### 1.1 当前书籍创建流程
- 目前的书籍创建流程是通过 `PipelineRunner.initBook` 方法实现的
- 流程包括：生成基础设定 → 审核基础设定 → 保存书籍配置 → 写入基础设定文件 → 初始化控制文档 → 创建初始快照
- 生成的基础设定包括：story_bible.md、volume_outline.md、book_rules.md、current_state.md、pending_hooks.md
- 目前没有明确的用户确认步骤，生成后会直接完成创建过程

### 1.2 当前书籍界面
- 目前的书籍详情页面主要显示：书籍基本信息、章节列表、设置选项
- 卷纲重生成功能已实现，但没有专门的卷纲和章节规划面板
- 章节操作主要通过下拉菜单实现，没有分卷查看功能

### 1.3 当前重写功能
- 已实现卷纲重生成功能，但没有分卷章节重写功能
- 重写后没有自动标记受影响章节的状态
- 没有一键重写本卷章节的功能

## 2. 优化目标

### 2.1 书籍创建流程优化
- 创建书籍时生成设定文件
- 根据设定生成书籍大纲和分卷卷纲
- 生成后等待用户确认卷纲
- 确认后手动执行生成分卷的章节规划

### 2.2 界面优化
- 增加卷纲和章节规划面板
- 可分卷查看章节和分卷规划内容
- 将重写卷纲功能相关操作按钮放入此界面

### 2.3 功能优化
- 卷纲和章节规划重写完成后标记关联的所有受影响章节状态
- 对相关章节重新审计
- 在分卷页面直接按钮一键重写本卷章节

## 3. 实现计划

### 3.1 后端修改

#### 3.1.1 PipelineRunner 优化
- 修改 `initBook` 方法，添加卷纲确认步骤
- 添加 `generateChapterPlansForVolume` 方法，用于生成分卷的章节规划
- 添加 `regenerateVolumeChapters` 方法，用于一键重写本卷章节
- 添加 `markAffectedChapters` 方法，用于标记受影响章节状态

#### 3.1.2 API 端点添加
- 添加 `/api/books/:id/volume-plans` 端点，用于获取卷纲和章节规划
- 添加 `/api/books/:id/volumes/:volumeId/generate-plans` 端点，用于生成分卷章节规划
- 添加 `/api/books/:id/volumes/:volumeId/rewrite-chapters` 端点，用于一键重写本卷章节

### 3.2 前端修改

#### 3.2.1 BookDetail.tsx 优化
- 添加卷纲和章节规划面板
- 实现分卷查看功能
- 添加卷纲确认界面
- 添加分卷章节规划生成按钮
- 添加分卷章节重写按钮

#### 3.2.2 新增 VolumeDetail.tsx
- 创建分卷详情页面，显示分卷信息、章节列表、章节规划
- 添加分卷章节重写功能
- 添加分卷章节审计功能

### 3.3 数据流优化

#### 3.3.1 书籍创建流程
1. 用户创建书籍，输入基本信息
2. 系统生成设定文件（story_bible.md、book_rules.md等）
3. 系统根据设定生成书籍大纲和分卷卷纲
4. 显示卷纲预览，等待用户确认
5. 用户确认后，系统生成分卷的章节规划
6. 书籍创建完成

#### 3.3.2 卷纲重写流程
1. 用户点击重写卷纲按钮
2. 系统生成新的卷纲
3. 显示卷纲预览，等待用户确认
4. 用户确认后，系统更新卷纲
5. 系统标记受影响章节状态
6. 系统对受影响章节重新审计

#### 3.3.3 分卷章节重写流程
1. 用户在分卷页面点击一键重写按钮
2. 系统重写本卷所有章节
3. 系统更新章节状态
4. 系统对重写的章节重新审计

## 4. 技术实现细节

### 4.1 后端实现

#### 4.1.1 PipelineRunner 修改
- 在 `initBook` 方法中添加卷纲确认逻辑

- 实现 `regenerateVolumeChapters` 方法，批量重写本卷章节
- 实现 `markAffectedChapters` 方法，根据卷纲变化标记受影响章节

#### 4.1.2 API 实现
- 在 server.ts 中添加新的 API 端点
- 实现卷纲和章节规划的获取、生成和重写功能
- 实现分卷章节重写功能

### 4.2 前端实现

#### 4.2.1 BookDetail.tsx 修改
- 添加卷纲和章节规划面板组件
- 实现分卷选项卡，可切换不同分卷
- 添加卷纲确认模态框
- 添加分卷章节规划生成按钮
- 添加分卷章节重写按钮

#### 4.2.2 VolumeDetail.tsx 实现
- 创建分卷详情页面，显示分卷信息
- 显示分卷包含的章节列表
- 显示分卷的章节规划
- 添加一键重写本卷章节按钮
- 添加章节审计按钮

### 4.3 数据流管理

#### 4.3.1 状态管理
- 使用 React useState 管理卷纲和章节规划状态
- 使用 React useEffect 处理数据加载
- 使用 SSE 处理实时更新

#### 4.3.2 数据持久化
- 卷纲和章节规划保存到对应的 markdown 文件
- 章节状态更新保存到 chapter_index.json
- 审计结果保存到章节元数据

## 5. 风险与应对措施

### 5.1 风险
- 卷纲重写可能导致情节不一致
- 章节规划生成可能与现有章节冲突
- 一键重写本卷章节可能导致大量内容变化

### 5.2 应对措施
- 实现重写幅度参数，控制原情节内容的保留幅度
- 在生成章节规划前检查与现有章节的冲突
- 在一键重写前显示确认提示，告知用户可能的影响
- 实现章节状态标记，便于用户识别受影响的章节
