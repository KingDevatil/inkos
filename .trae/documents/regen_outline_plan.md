# 卷纲重新生成与章节规划更新实现计划

## 1. 需求分析

用户希望能够：
- 通过输入框输入作者意图，重新修改生成卷纲
- 生成后等待用户确认
- 确认后根据新卷纲重新生成章节规划
- 保证新生成的卷纲、章纲与相邻的卷、章完美衔接

## 2. 现有系统分析

### 相关组件
- **ArchitectAgent**：负责生成基础设定和卷纲
- **PlannerAgent**：负责生成章节规划
- **PipelineRunner**：管理整个写作流程
- **StateManager**：管理书籍状态和文件

### 现有文件结构
- `story/volume_outline.md`：存储卷纲
- `story/author_intent.md`：存储作者意图
- `story/runtime/chapter-XXXX.intent.md`：存储章节规划

### 现有流程
1. 书籍创建时，ArchitectAgent 生成初始卷纲
2. 每章写作前，PlannerAgent 基于卷纲生成章节规划
3. 卷纲一旦生成，没有直接的更新机制

## 3. 实现计划

### 3.1 后端实现

#### 1. 扩展 ArchitectAgent
- 添加 `regenerateOutline` 方法，支持基于作者意图重新生成卷纲
- 该方法应接收书籍配置、作者意图和重写幅度参数
- 生成新卷纲时，应考虑相邻卷的内容，确保卷与卷之间的衔接
- 基于重写幅度参数，控制原情节内容的保留幅度与重写幅度

#### 2. 扩展 PipelineRunner
- 添加 `regenerateOutline` 方法，暴露给 API 和 CLI
- 实现卷纲生成、预览和确认的完整流程
- 添加 `preGenerateChapterPlans` 方法，根据新卷纲提前生成多个章节的规划
- 添加 `updateChapterPlans` 方法，根据新卷纲更新章节规划

#### 3. 扩展 API 端点
- 添加 `POST /api/books/:id/regenerate-outline` 端点
- 支持接收作者意图和重写幅度参数
- 返回生成的新卷纲供用户确认
- 添加 `POST /api/books/:id/confirm-outline` 端点
- 接收用户确认，更新卷纲并重新生成章节规划

#### 4. 扩展 CLI 命令
- 添加 `inkos book regenerate-outline` 命令
- 支持 `--intent` 参数输入作者意图
- 支持 `--rewrite-level` 参数控制重写幅度
- 提供卷纲预览和确认机制

### 3.2 前端实现

#### 1. 扩展 BookDetail 页面
- 添加卷纲重新生成按钮
- 实现作者意图输入表单
- 添加重写幅度滑块或选择器
- 显示生成的新卷纲预览
- 提供确认和取消按钮
- 实现更新进度和结果反馈

#### 2. 状态管理
- 处理卷纲生成、预览、确认的状态流转
- 显示加载状态和错误处理

### 3.3 衔接性保证

#### 1. 卷纲衔接策略
- 分析相邻卷的内容，提取关键情节节点
- 新卷纲生成时，确保与上一卷和下一卷的情节逻辑一致
- 基于重写幅度参数，控制原情节内容的保留程度

#### 2. 章节规划更新
- 基于新卷纲，提前生成多个章节的规划，确保节奏一致
- 确保章节规划与新卷纲的节点对应
- 确保章节规划与相邻章节的内容衔接

## 4. 技术实现细节

### 4.1 ArchitectAgent 扩展
- 复用现有的 `generateFoundation` 方法逻辑
- 添加上下文感知，分析相邻卷的内容
- 生成卷纲时，确保与相邻卷的衔接
- 实现重写幅度参数的处理逻辑

### 4.2 卷纲预览机制
- 生成新卷纲后，不直接覆盖原有文件
- 存储为临时文件，供用户预览
- 用户确认后，才覆盖原卷纲文件

### 4.3 章节规划更新
- 识别受新卷纲影响的章节范围
- 提前生成多个章节的规划，确保节奏一致
- 确保章节规划与新卷纲的情节节点对应
- 确保章节规划与相邻章节的内容衔接

## 5. 风险与应对策略

### 5.1 风险
- 新卷纲可能与相邻卷内容冲突
- 用户可能不满意生成的卷纲
- 章节规划更新可能影响已开始写作的章节

### 5.2 应对策略
- 提供详细的卷纲预览，让用户充分了解变更
- 保留原卷纲的备份，允许用户回滚
- 对已开始写作的章节，提供智能衔接建议
- 通过重写幅度参数，让用户控制变更程度

## 6. 实施步骤

1. **扩展 ArchitectAgent**：添加 `regenerateOutline` 方法，支持重写幅度参数
2. **扩展 PipelineRunner**：添加卷纲更新和章节规划更新方法，支持提前生成章节规划
3. **扩展 API 端点**：添加卷纲重新生成和确认端点，支持重写幅度参数
4. **扩展 CLI 命令**：添加卷纲重新生成命令，支持重写幅度参数
5. **扩展前端界面**：添加卷纲重新生成功能，包括重写幅度控制
6. **测试与验证**：确保功能正常运行，衔接性良好

## 7. 预期效果

- 用户可以通过输入框输入作者意图，设置重写幅度，触发卷纲重新生成
- 系统生成新卷纲并提供预览
- 用户确认后，系统更新卷纲并提前生成多个章节的规划
- 新生成的卷纲和章纲与相邻的卷、章完美衔接
- 整个过程用户友好，提供清晰的反馈和控制
