# InkOS Agent Skill

通过自然语言指令调用 InkOS Agent API，让 AI 自动协调执行各种书籍创作和管理任务。

## 功能概述

InkOS Agent 是一个基于 LLM 的工具使用代理，可以理解自然语言指令并自动调用相应的工具来完成任务。支持的操作包括：

- 书籍管理（创建、更新、删除）
- 章节写作（起草、重写、续写）
- 审核与修订
- 题材管理
- 导出与导入
- 状态查询与分析

## 使用方法

### 基本调用

当用户想要通过自然语言控制 InkOS 时，使用此 Skill。

**示例指令：**
- "帮我写下一章"
- "审核最近写的章节"
- "创建一个修仙题材的新书"
- "导出已完成的书籍"
- "检查书籍状态"

### API 端点

```
POST /api/agent
```

### 请求参数

```typescript
{
  instruction: string;  // 自然语言指令（必需）
}
```

### 响应格式

```typescript
{
  response: string;  // Agent 的执行结果或回复
}
```

## 使用示例

### 示例 1：写作指令

```javascript
// 请求
POST /api/agent
{
  "instruction": "帮我写下一章，主角要在这一章突破到金丹期"
}

// 响应
{
  "response": "已完成第15章的写作。本章描述了主角在闭关三个月后，成功突破到金丹期的过程..."
}
```

### 示例 2：审核指令

```javascript
// 请求
POST /api/agent
{
  "instruction": "审核最近3章的内容"
}

// 响应
{
  "response": "已完成第12-14章的审核。发现以下问题：\n1. 第12章：时间线不一致...\n2. 第13章：人物设定冲突..."
}
```

### 示例 3：创建书籍

```javascript
// 请求
POST /api/agent
{
  "instruction": "创建一本新书，名字叫《修仙从杂役开始》，题材选修仙，平台选番茄"
}

// 响应
{
  "response": "已成功创建书籍《修仙从杂役开始》(xiu-xian-cong-za-yi-kai-shi)。\n- 题材：修仙\n- 平台：番茄\n- 目标章节：100章\n- 每章字数：3000字"
}
```

## 注意事项

1. **指令清晰性**：尽量使用清晰、具体的指令，有助于 Agent 更准确地理解意图
2. **上下文信息**：可以在指令中包含书籍 ID、章节号等具体信息
3. **异步操作**：写作、审核等操作可能需要较长时间，会通过 SSE 事件流推送进度
4. **错误处理**：如果指令无法执行，Agent 会返回错误说明

## 相关事件 (SSE)

Agent 操作期间会触发以下事件：

- `agent:start` - Agent 开始执行
- `agent:complete` - Agent 执行完成
- `agent:error` - Agent 执行出错
- `write:start` / `write:complete` / `write:error` - 写作相关
- `audit:start` / `audit:complete` / `audit:error` - 审核相关

## 与其他 API 的关系

Agent API 是对底层 API 的高级封装：
- Agent 内部会调用 `/api/books/:id/write-next` 等具体端点
- 适合复杂的多步骤任务
- 对于简单的单步操作，可以直接调用具体 API
